\f1\b\fs28 \cf2 Why custom OAuth instead of Supabase Auth\
\pard\pardeftab720\partightenfactor0

\f0\b0\fs24 \cf2 Supabase Auth would only simplify the 
\f2\i login
\f0\i0  half of the problem. The harder half \'97 obtaining and refreshing a Google 
\f1\b Drive
\f0\b0  refresh token for server-side uploads to the user's Drive folder \'97 is not something Supabase Auth manages for you. Given Supabase is scoped down to "just a database" for this build, running Auth through Supabase in addition would mean verifying Supabase-issued JWTs against Supabase's JWKS endpoint from an Express backend that owns everything else \'97 an extra integration seam for no real benefit. A single, self-owned OAuth implementation handles both login and Drive access through the same token machinery, and keeps one system of record for identity.\
\pard\pardeftab720\partightenfactor0
